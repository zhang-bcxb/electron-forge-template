## 项目说明

- 项目运行

```
安装依赖：npm i
运行项目：npm run start
```

- 打包说明

```
"package" 命令： 这个命令用于将你的 Electron 应用程序打包为一个发布版本，生成一个可分发的应用程序包。
这个包包含了应用程序的所有代码和资源，但还没有经过任何安装步骤。
通过运行 "electron-forge package" 命令，你可以创建一个用于分发和安装的打包文件。

"make" 命令： 一旦应用程序被打包，"make" 命令用于将这个打包好的应用程序包转化为不同操作系统的可执行文件。
例如，它可以将你的应用程序打包成可在 Windows、macOS 和 Linux 上运行的可执行文件。
这一步包括为不同平台生成可执行文件，并根据操作系统的要求进行相应的配置。

"publish" 命令： 这个命令用于将打包好的、可执行的应用程序发布到各种应用商店或发布平台。
这可以包括将应用程序发布到各个平台的应用商店
（如 Windows Store、Mac App Store、Linux Software Center）或通过网络发布给用户下载安装。
```

## 项目经验

- 窗口实现透明效果

```
// 样式设置成半透明
background-color: rgba(0, 0, 0, 0.5);

// 设置窗口的外框（标题栏和窗口边框）不显示，使窗口成为一个无边框窗口。
frame: false,
// 设置窗口背景为透明，使窗口的内容可以透过窗口看到后面的内容
transparent: true,
```

- 发送消息的区别

```
browserWindow.webContents.send('消息名称')：
这个代码用于在 Electron 的主进程中向指定的渲染进程发送消息。
具体来说，它会将一个名为 '消息名称' 的消息发送给指定的 BrowserWindow 的渲染进程。
在渲染进程中，你可以使用 ipcRenderer 监听这个消息，然后执行相应的操作。
通常情况下，这种通信方式用于在渲染进程中触发某个特定的操作，例如在用户点击菜单项时触发创建新文件的操作。
总结：在渲染进程中处理消息

ipcRenderer.send('消息名称')：
这个代码用于在 Electron 的渲染进程中向主进程发送消息。
具体来说，它会将一个名为 '消息名称' 的消息发送给主进程。
在主进程中，你可以使用 ipcMain 监听这个消息，然后执行相应的操作。
这种通信方式用于将渲染进程中发生的事件或用户操作通知给主进程，然后主进程可以执行与之相关的操作。
例如，当用户在渲染进程中拖动窗口时，可以使用这种通信方式来告知主进程窗口的新位置。
总结：在主进程中处理消息
```
